Add simple evaluation task implement

This PR introduces evaluation support designed to track and benchmark applications built on the FastGPT platform.

Key changes:
- Adds an evaluation task framework focused on app-level tracking and benchmarking for FastGPT-built apps.
- Enables defining evaluation scenarios, running evaluations, collecting metrics, and persisting results for reproducible benchmarks.
- Integrates simple logging and result storage to record run metadata (app id/version, environment, timestamps).
- Changes: 28 files, +1455 additions, -66 deletions.
- Branch: add-evaluations -> main.

Basic evaluation logic:
- Evaluation definitions: specify metrics, datasets/input cases, app identifiers, and run configuration.
- Runner: executes evaluation tasks against app endpoints or model components, captures responses and timing, and computes metrics such as accuracy, latency, and success rate.
- Metrics & aggregation: supports per-case metrics and aggregated summaries for per-app and cross-app benchmarking; custom metric plugins are supported.
- Persistence & logging: stores run results, metadata, and logs for reproducibility and downstream analysis.
- Reporting: produces aggregate scores and per-run/per-app breakdowns to support benchmarking and comparison.
- Extensibility: designed to allow additional metrics, evaluators, datasets, dashboards, and CI integration.

Motivation:
Providing evaluation capabilities tailored to apps built on FastGPT enables developers and teams to track performance over time, compare app versions, and benchmark implementations across deployments.

Next steps:
- Expand metrics (e.g., F1, BLEU, ROUGE), dataset integrations, and evaluation scenarios.
- Add dashboards and CI hooks for automated benchmarking.
- Improve documentation and examples for integrating evaluations into apps.
